## 1. Суми замовлень у щоденному звіті

Було: `→ 2 замовлення (10%)`.
Стало: `→ 2 зам. на 6 010 ₴ (10%)` + рядок вгорі "Всього замовлень: 39 | на суму 62 906 ₴".

Сортування у звіті — за сумою (найприбутковіші реклами зверху), а не за кількістю. Відсоток — частка у загальному доході.

**Зміна:** `src/analyzer/ad_analytics.py`:
- `build_ad_report()` тепер повертає ще `sums`, `total_sum` (по `totalPriceDiscount`)
- `format_ad_report()` — новий формат, сортує за сумою

## 2. Google Sheet "Ефективність реклами 2"

### Мета
Накопичувальна ROI-таблиця. Витрати на рекламу заповнюються вручну, а бот щодня додає суму замовлень по кожному креативу.

### Структура
- **Колонка A** — `adTitle` з Sitniks (текстова назва креативу)
- **Колонка B** — `=SUM(...)` формула, підсумовує всі дати
- **C**: Січень (label), **D..AH**: 01.01–31.01, **AI**: Лютий... — всього 12 місяців × 32 колонки = 386 колонок

### Автозаповнення
`src/sheets/ads_sums.py` — новий модуль. Основна функція:

```python
async def write_daily_sums_to_sheet(target_date):
    # 1. build_ad_report() для target_date
    # 2. Для кожної нової реклами додає рядок у кінець
    # 3. У колонку відповідної дати пише суму
    # 4. Обнуляє клітинки, у яких раніше було значення,
    #    але у свіжому звіті реклами вже немає (потрібно для reattribute)
```

### Cron
- Спочатку окремий job о **08:45** — але виявилося, він **завжди фейлився через rate-limit** Sitniks (див. розділ 4)
- Виправлення: `write_daily_sums_to_sheet` **інлайн** у кінці `send_daily_ads_report` (о 08:30) → один потік, немає паралельних запитів
- Також викликається у `reattribute_yesterday` (22:00) — якщо нічна корекція знайде розбіжності

Коміти: `a7870ad` (перша версія), `7b2b2c4` (інтеграція в 08:30)

## 4. Виявлені баги

### 4.1. Scheduler «мертвий» після рестарту
**Симптом:** cron-job'и не спрацьовували ранком, хоча контейнер живий.
**Причина:** apscheduler `AsyncIOScheduler` + 3 aiogram polling-и в одному event-loop → полінг ловить `TelegramConflictError` циклами → event loop starvation → scheduler не тікає.
**Виправлення:**
- `misfire_grace_time` 3 год → **12 год** (щоб пропущений job спрацював пізніше, а не пропускався зовсім)
- Додано **heartbeat** кожні 30 хв — щоб бачити чи планувальник живий
- Улучшене логування — при старті виводить усі job-и з часом наступного запуску

Коміти: `acd7768`, `a1171e4`

### 4.2. Конкуренція за Sitniks API rate-limit
**Симптом:** `daily_ads_sheet_job` о 08:45 ніколи не завершувався, попри те що `daily_ads_report` о 08:30 працював.
**Причина:** обидва job'и інтенсивно тягнуть Sitniks. `send_daily_ads_report` виконується 5–10 хв, і поки він триває, `daily_ads_sheet_job` о 08:45 стартує паралельно → 429 rate-limit → безкінечні ретраї → job вмирає непомітно.
**Виправлення:** об'єднав обидва в один — `write_daily_sums_to_sheet` викликається **інлайн** після `send_daily_ads_report`. Окремий cron видалений.

Коміт: `7b2b2c4`

### 4.3. Railway trial expired
**Симптом:** контейнер зупинено 21:33 Київ 10.07, не перезапустився → всі job'и 11.07 не спрацювали.
**Причина:** `Your trial has expired. Please select a plan to continue using Railway.`
**Виправлення:** користувач обрав платний план ($5/міс Hobby).

### 4.4. Google Sheets credentials відсутні на Railway
**Симптом:** `daily_ads_sheet` (і `daily_hair_stats`) з середини червня фейляться на Railway з:
```
[Errno 2] No such file or directory: '/app/google-service-account.json'
```
**Причина:** файл `google-service-account.json` у `.gitignore`, тобто не потрапляє в Docker-контейнер. Env-змінна `GOOGLE_SERVICE_ACCOUNT_B64` (fallback у коді) не задана на Railway.
**Ілюзія «воно працює»:** щоденні suma-записи виконувалися **вручну з локального лаптопа** (де файл є) — не автоматично.
**Виправлення:** потрібно додати `GOOGLE_SERVICE_ACCOUNT_B64` через Railway UI (Claude Code auto-mode блокує upload приватного ключа через API). Буде зроблено пізніше.
